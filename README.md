# Wordstat Дашборд — статистика ключевых слов по регионам РФ

Минималистичный веб-дашборд для анализа статистики поисковых запросов через
**Yandex Wordstat API** ([документация](https://aistudio.yandex.ru/docs/ru/search-api/concepts/wordstat.html)).

## Возможности

- Поиск статистики по ключевой фразе.
- **Строгая разбивка по самостоятельным субъектам РФ.** Федеральные округа,
  Россия целиком и зарубежные регионы полностью исключаются из данных.
- **Рынок и бренд ОТП в одном экране.** Для введённой фразы всегда строятся два
  среза: «рынок» (ваша фраза) и «ОТП» (та же фраза со словом **ОТП**). Карточка
  показывает суммарную **долю ОТП от общих запросов**.
- **Строгая разбивка по самостоятельным субъектам РФ.** Федеральные округа,
  Россия целиком и зарубежные регионы полностью исключаются из данных.
- Аналитические графики (Chart.js):
  - распределение по регионам — **два столбца на регион** (рынок + ОТП), топ-15 / 30;
    в подсказке — абсолют и доля ОТП от рынка;
  - **динамика частоты с двумя осями ординат**: ОТП (левая ось) и рынок (правая),
    с выбором региона и периода (**месяц / неделя / день**);
  - **гистограмма распределения числа запросов** — сколько регионов попадает в
    каждый диапазон показов;
  - **две кольцевые диаграммы** долей регионов — отдельно ОТП и отдельно рынок,
    с агрегатом «Остальные регионы» (всё, кроме топ-10).
- Таблица регионов с фильтром: рынок, ОТП, доля ОТП, индекс соответствия.
- **Сравнение нескольких фраз** (2–5) на одном графике динамики: суммы, доли и
  темп роста по каждой фразе + отдельная выгрузка в Excel. Известные продукты
  показывают реальные данные из CSV даже без API.
- **Выгрузка в Excel** (.xlsx): лист «Регионы РФ» (рынок, ОТП, доля ОТП, доля
  рынка, индекс + помесячная разбивка) и лист «Динамика» (рынок и ОТП).
- **Премиальный интерфейс в фирменном стиле ОТП**: салатовый (рынок), фиолетовый
  (ОТП) и оранжевый (акцент) — читаемая семантика цвета вместо «всё зелёное».
- Рабочие селекторы **устройств** (масштабируют объёмы) и **периода динамики**.
- **Демо-режим**: без ключа API показываются синтетические данные —
  интерфейс и выгрузка работают сразу.

## Запуск

Windows — двойной клик по `run.bat`, либо вручную:

```powershell
python -m pip install -r requirements.txt
python app.py
```

Откройте <http://127.0.0.1:5000>.

> **macOS:** порт 5000 по умолчанию занимает AirPlay. Запустите на другом порту:
> `python app.py --port 5050` (или `PORT=5050 python app.py`) и откройте
> <http://127.0.0.1:5050>.

## Рабочий режим (реальный API)

1. Скопируйте `.env.example` в `.env`.
2. Заполните `YANDEX_API_KEY` (или `YANDEX_IAM_TOKEN`) и `YANDEX_FOLDER_ID`.
3. Перезапустите сервер. Бейдж в правом верхнем углу сменится на «рабочий режим».

Требуется роль `search-api.webSearch.user` в Yandex Cloud.

### Если на дашборде синтетические данные (API отвечает 403)

Если бейдж показывает «рабочий режим», но цифры по незнакомым фразам
выглядят синтетическими — значит вызовы к Wordstat падают с
`403 Permission denied`, и приложение прозрачно откатывается на локальные
данные (CSV по продуктам + модель). Проверьте по порядку:

1. **Роль сервис-аккаунта.** Аккаунту, которому принадлежит API-ключ, нужна
   роль `search-api.webSearch.user` **на том же каталоге**, что указан в
   `YANDEX_FOLDER_ID` (Console → IAM → права доступа каталога).
2. **Каталог.** `YANDEX_FOLDER_ID` должен принадлежать тому же облаку, где
   создан сервис-аккаунт/ключ.
3. **Биллинг.** К облаку привязан активный платёжный аккаунт (триал не истёк).
4. **Search API** включён для каталога.

Текст ошибки можно увидеть так (подставьте свои ключ и каталог):

```bash
curl -s -X POST https://searchapi.api.cloud.yandex.net/v2/wordstat/regions \
  -H "Authorization: Api-Key $YANDEX_API_KEY" -H "Content-Type: application/json" \
  -d '{"phrase":"кредит наличными","region":"REGION_REGIONS","folderId":"'"$YANDEX_FOLDER_ID"'"}'
```

Известные продукты (см. `data/search_queries_v3.csv`) показывают **реальные**
данные даже без API — они берутся из ручной выгрузки Wordstat.

## Защита входа (логин/пароль)

Публичный сайт по умолчанию **открыт всем**. Чтобы закрыть его, задайте в
`.env` пароль (HTTP Basic Auth защищает и страницу, и API):

```bash
APP_USER=otp          # логин (по умолчанию "otp")
APP_PASSWORD=сильный-пароль
```

Без `APP_PASSWORD` защита выключена. При деплое можно передать сразу:

```bash
APP_PASSWORD='секрет' YANDEX_API_KEY=AQVN... YANDEX_FOLDER_ID=b1g... \
  bash -c 'curl -fsSL https://raw.githubusercontent.com/MAKAR2007/scraping_wordstat/main/deploy/setup.sh | bash'
```

> ⚠️ На обычном HTTP пароль передаётся в base64 (не шифруется). Для публичного
> сайта включайте Basic Auth **вместе с HTTPS** — тогда трафик зашифрован.

## Деплой на свой VPS (Ubuntu 22.04/24.04)

Один скрипт ставит Python/nginx, поднимает gunicorn как systemd-сервис и
проксирует на порт 80. Запускать от root:

```bash
# демо-режим (реальные данные продуктов из CSV):
curl -fsSL https://raw.githubusercontent.com/MAKAR2007/scraping_wordstat/main/deploy/setup.sh | bash

# рабочий режим (живой API) — передайте креды переменными:
YANDEX_API_KEY=AQVN... YANDEX_FOLDER_ID=b1g... \
  bash -c 'curl -fsSL https://raw.githubusercontent.com/MAKAR2007/scraping_wordstat/main/deploy/setup.sh | bash'
```

Сайт открывается по `http://<IP-сервера>/` — **без VPN, в том числе из РФ**:
весь фронтенд (Chart.js, шрифты) отдаётся с самого VPS, обращений к сторонним
CDN нет. Установщик **идемпотентен**: повторный запуск = обновление до свежего
кода (`git reset --hard origin/main`), при этом `.env` сохраняется.

Полезное на сервере:

```bash
systemctl status wordstat      # состояние сервиса
journalctl -u wordstat -f      # логи приложения
systemctl restart wordstat     # перезапуск после правки .env
```

## Как обеспечивается исключение федеральных округов

В дереве регионов Wordstat структура такая:

```
Россия (225)
  └── Федеральный округ      ← агрегат, ИСКЛЮЧАЕТСЯ
        └── Субъект РФ        ← ОСТАВЛЯЕМ (область / край / республика / город ФЗ / АО)
```

Модуль [`regions.py`](regions.py) берёт прямых потомков узла «Россия» как
федеральные округа (исключает их) и оставляет только их потомков — субъекты
РФ. Фильтр `filter_to_subjects` отбрасывает любые строки ответа API, чьи id
не входят в список субъектов: тем самым округа, Россия целиком и зарубежные
регионы гарантированно не попадают ни в графики, ни в Excel.
